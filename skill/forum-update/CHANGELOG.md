# Forum Update — журнал изменений

Версионность: семантическая (`MAJOR.MINOR.PATCH`).
- **MAJOR** — несовместимые изменения методологии (например, отказ от правила 5% или смена базовой структуры апдейта).
- **MINOR** — новая секция или существенное переоформление без слома совместимости (старые апдейты остаются валидными по новым критериям).
- **PATCH** — формулировки, опечатки, мелкие уточнения, починка ссылок.

Текущая версия — в `SKILL.md` → frontmatter поле `version`.

Upstream: [github.com/dmitry626/forum-update-kit](https://github.com/dmitry626/forum-update-kit). Локальные правки, не доведённые до upstream, помечаются в записи как `[local]`.

## 1.1.0 — 2026-05-22

Переоформлена работа с запросом — приведена в соответствие с учебником модератора форум-группы (CMS, Я. Агарунова: статьи «13 Парковка и выбор темы», «14 Презентация запроса»). Старые апдейты совместимы — критерии не отменены, переформулированы.

**Источник правки.** В сессии подготовки апдейта 22.05 навык предложил варианты запроса в форме «что во мне X», «от чего меня защищает Y», «почему я делаю Z». Это формулировки self-inquiry, не форумный запрос. Учебник прямо называет презентацию «**не** психологической сессией»: контракт презентации — обмен опытом, не разбор презентера. Расхождение исправлено.

**Файлы:**

- `methodology.md` — переписана секция «Запрос — самая важная часть»:
  - Критерий «Внутренний, не внешний» заменён на «Уязвимая тема + форма "как / что"». Тема приходит из интроспекции, форма — приглашает опыт группы.
  - Блок рефрейминга — примеры self-inquiry («что я теряю, когда выбираю X», «что я получаю от откладывания») заменены на конкретизацию темы через факт периода с сохранением формы «как / что» («как у вас получается X в фазе Y»).
  - Добавлена явная контрастная инструкция «что НЕ работает как форма запроса» — три антипаттерна: «что во мне», «от чего защищает», «почему я».
  - Добавлен раздел «Подготовка к запросу vs формулировка запроса — две разные стадии» — фиксация, что вопросы фазы 3.5 (избегаю, сопротивляюсь) — для нахождения темы наедине с собой, не для использования в запросе.
  - Добавлен третий критерий «Группа в запросе» (из учебника, 13 Парковка): запрос должен быть таким, чтобы у участников этой группы был релевантный опыт.
  - Обновлены «Красные / Зелёные флаги» — терминология «внутренний/внешний» снята.

- `SKILL.md` — обновлён режим «Чек», секция 6 «Запрос»:
  - Шаг 2 «Глубина»: критерий «внешний / внутренний» заменён на «форма приглашает опыт группы».
  - Шаг 3 «Рефрейминг»: вместо «внешнее → внутреннее» — «конкретизация темы через факт периода с сохранением формы как/что».
  - Добавлено поле `version` в frontmatter.

- `template-prep.md` — фаза 3.4 «Варианты запроса»: добавлен чек по 3 пунктам перед перечислением вариантов (форма / тема / отвечается опытом).

**Что не менялось:** правило 5%, 7 маркеров значимости, структура тезиса «факт + почему + чувства», словарь эмоций, тайминг 3-5 минут, баланс сфер, фаза 3.5 «Вопросы для самопроверки» (они изначально согласованы с учебником как подготовительные вопросы).

**Совместимость:** апдейты, написанные по версии 1.0.0, валидны по 1.1.0 — критерии не отменены, а переформулированы точнее.

## 1.0.0 — 2026-04 (исходная версия)

Первая версия навыка, опубликованная как часть `forum-update-kit` на GitHub.

Базовая методология:
- Правило 5%
- 7 маркеров значимости (идентичность / связь / тело / полярные эмоции / впервые / уязвимость / внутренний сдвиг)
- Структура тезиса «факт + почему + чувства»
- Три критерия запроса (внутренний / уязвимый / свежий)
- Словарь эмоций по NVC
- Шаблоны подготовки и финального апдейта
- Два режима навыка: prep и check
