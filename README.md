# Forum Update Kit

Набор промптов, шаблонов и чек-листа для подготовки и проверки форум-апдейта перед встречей бизнес-клуба.

Автор: Дмитрий ([@dmitry626](https://github.com/dmitry626))

---

## Зачем

Форум-апдейт — короткий личный монолог на встрече форум-группы: что значимого произошло за месяц, что даёт и забирает энергию, с каким запросом ты приходишь к группе. Хороший апдейт делает встречу глубокой. Поверхностный апдейт превращает форум в отчёт, в котором нет работы.

Большинство из нас готовит апдейты наспех. Формат знакомый, но дисциплина подготовки — нет. Этот набор помогает:

- **Проверить готовый апдейт перед встречей** — попал ли ты в 5% самого важного, не свалился ли в операционку, назвал ли настоящие чувства, уязвим ли запрос к группе.
- **Подготовиться осознанно** — пройти по структуре так, чтобы вытащить из себя 5%, а не отписаться.

Работает с любым AI (ChatGPT, Claude, Gemini). Если у тебя Claude Code — есть отдельная skill-форма.

## Как пользоваться — 3 режима

### Режим 1. Проверить готовый апдейт (любой AI)

Самый простой сценарий. Подходит, если ты уже написал апдейт и хочешь проверить его перед встречей.

1. Открой [`prompts/check.md`](prompts/check.md), скопируй всё содержимое.
2. Вставь в ChatGPT / Claude / Gemini.
3. После промпта вставь текст своего апдейта.
4. Получишь разбор: какие тезисы попадают в 5%, где чувств недостаточно, глубокий ли запрос, что стоит усилить.

### Режим 2. Подготовиться к апдейту (любой AI)

Если апдейта ещё нет, но хочешь помочь себе его написать.

1. Открой [`prompts/prep.md`](prompts/prep.md), скопируй всё содержимое.
2. Вставь в AI.
3. AI задаст тебе вопросы по 4 сферам (бизнес / семья / любовь / личное), поможет найти противоречия и сформулировать запрос.
4. Финальный апдейт пишешь сам по шаблону [`skill/forum-update/template-update.md`](skill/forum-update/template-update.md).

### Режим 3. Claude Code skill

Если ты пользуешься Claude Code и ведёшь заметки в Obsidian/markdown-vault, skill-форма даёт больше: читает твои дневники / ретроспективы / календарь / историю прошлых апдейтов и строит досье за период.

1. Скопируй папку `skill/forum-update/` в `~/.claude/skills/`:
   ```bash
   git clone https://github.com/dmitry626/forum-update-kit.git
   cp -r forum-update-kit/skill/forum-update ~/.claude/skills/
   ```
2. В Claude Code набирай `/forum-update` (подготовка) или `/forum-update check` (проверка).

Skill рассчитан на vault с дневником и ретроспективами, но адаптируется под любую структуру — методология остаётся та же.

## Три опоры методологии

1. **Правило 5%.** В апдейт идут только 5% самого значимого — самые тяжёлые и самые радостные события. Остальные 90% — для коллег, друзей, семьи. Форум нужен только для того, чего не произнесёшь в обычном разговоре.

2. **Структура на каждый тезис: факт + почему важно + чувства.** Минимум три конкретных чувства на блок — не «хорошо / плохо», а точные слова из словаря эмоций.

3. **Запрос — внутренний, уязвимый, свежий.** Не «как сделать X» (на это даёт совет любой деловой разговор), а «чего я избегаю», «почему знаю и не делаю», «что я не вижу в себе». Запрос, который стыдно произнести — правильный запрос.

Подробная методология: [`skill/forum-update/methodology.md`](skill/forum-update/methodology.md).

## Disclaimer

- Это **мой личный чек-лист**, не единственно правильный формат. Разные форум-группы готовят апдейты по-разному, и в этом есть ценность.
- Набор **не заменяет модератора**. Модератор ведёт встречу, держит пространство, принимает решения по парковке и выбору темы. Этот набор — только про твою личную подготовку.
- Методология правила 5%, структуры апдейта и парковки взята из практики форум-групп. За полной методологией модерации — к учебнику модератора в своём клубе.
- AI ошибается. Если разбор кажется неточным — доверяй себе, не модели.
